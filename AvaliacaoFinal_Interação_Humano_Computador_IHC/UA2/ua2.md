# UA2 – Interface, interação e Affordance

## Objetivo
Investigar como diferentes modalidades de interação e restrições físicas impactam a experiência do usuário, propondo soluções multimodais que garantam inclusão e autonomia para usuários com necessidades específicas, especialmente com deficiência visual.

## Desafio Proposto
O desafio consistiu em identificar o tipo de interface e os equipamentos mais adequados para um sistema utilizado por funcionários com deficiência visual (como o caso de um gerenciador de cursos ou sistema de agendamento de consultas médicas), justificando a escolha com base nos princípios de acessibilidade e IHC.

## Solução Desenvolvida e Justificativa
Desenvolveu-se uma análise aprofundada de um sistema voltado para usuários com deficiência visual. A pesquisa identificou que a interface tradicional baseada em mouse e tela não atendia adequadamente a este público, eliminando a dependência de elementos gráficos.

### 1. Tipo de Interface Proposta
Foi proposta uma combinação de **Interface de Voz (VUI — Voice User Interface)** e **Interface de Usuário de Áudio (AUI)**, com suporte nativo a leitores de tela. Essa abordagem permite que toda a interação e o retorno de informações ocorram de forma sonora e acessível, garantindo autonomia plena.

### 2. Equipamentos e Tecnologias Assistivas (Hardware e Software)
Para o melhor funcionamento da interface, o ecossistema recomendado inclui:
- **Softwares Leitores de Tela:** NVDA (NonVisual Desktop Access) ou JAWS, fundamentais para traduzir o conteúdo textual em áudio.
- **Dispositivos de Áudio:** Fones de ouvido de boa qualidade para audição clara do retorno do sistema e microfone para comandos de voz.
- **Hardware de Entrada:** Teclado físico padrão (com atalhos customizáveis e feedback tátil) em vez de mouses.
- **Dispositivos Opcionais:** Displays ou linhas Braille, que oferecem uma alternativa tátil para leitura e conferência de dados precisos.

### 3. Benefícios Alcançados
A solução proposta integrou múltiplas affordances que permitem aos usuários com deficiência visual ouvir o que está sendo exibido, verificar com precisão o que foi digitado e entender quais objetos estão manipulando (botões, formulários e menus). Isso elimina barreiras operacionais e alinha o sistema ao princípio "Perceptível" das diretrizes WCAG 2.1.
